# P1309 最长单调递增子序列

#### 题目描述

> 求一个字符串的最长递增子序列的长度

> 如：dabdbf 最长递增子序列就是 abdf，长度为 4

#### 输入描述

> 第一行一个整数 0<n<20, 表示有 n 个字符串要处理

> 随后的 n 行，每行有一个字符串，该字符串的长度不会超过 10000

#### 输出描述

> 输出字符串的最长递增子序列的长度

#### 样例输入

> 3

> aaa

> ababc

> abklmncdefg

#### 样例输出

> 1

> 3

> 7

#### 提示

本题有多组输入，请使用多组输入形式读入


------
# 题解

## 算法核心：贪心 + 二分查找

### 思路

我们维护一个数组 `d`，其中 `d[i]` 表示**长度为 i+1 的递增子序列的末尾元素的最小可能值**。遍历原序列的每个字符 `c`：

- 在 `d` 中**二分查找第一个大于等于 `c` 的位置**（因为我们要严格递增，所以用 `lower_bound`）。
- 如果找不到（即所有元素都小于 `c`），说明 `c` 可以接在最长子序列后面，将 `c` 追加到 `d` 末尾。
- 如果找到了位置 `p`，说明存在一个长度为 `p+1` 的子序列，其末尾可以更新为更小的 `c`（因为原来 `d[p] >= c`，现在用 `c` 替换后，后续更容易接上更大的数）。

最终 `d` 的长度就是最长递增子序列的长度。

### 为什么这样正确？

- `d` 数组是单调递增的（因为每个位置保存的是该长度下最小的末尾值）。
- 每次替换不会改变长度的正确性，但为后续提供了更优的末尾值。
- 这是一种典型的“耐心排序”思想。

### 举例

以 `"ababc"` 为例：

- 遍历 a: d = [a]
- b: lower_bound([a], b) 得到位置1（没有>=b，所以追加）→ d = [a, b]
- a: lower_bound([a,b], a) 得到位置0（因为a>=a），替换 → d = [a, b]（实际上a替换了a，没变化）
- b: lower_bound([a,b], b) 得到位置1（b>=b），替换 → d = [a, b]
- c: lower_bound([a,b], c) 得到位置2（没有>=c），追加 → d = [a, b, c]
  长度为3，正确。

## 代码实现（C++）

```cpp
#include <iostream>
#include <vector>
#include <string>
#include <algorithm>
using namespace std;

// 计算一个字符串的最长严格递增子序列长度
int LIS(const string& s) {
    vector<char> d;  // d[i] 表示长度为 i+1 的递增子序列的最小末尾字符
    for (char c : s) {
        // 二分查找第一个大于等于 c 的位置
        auto it = lower_bound(d.begin(), d.end(), c);
        if (it == d.end()) {
            // c 比所有末尾都大，可以扩展最长长度
            d.push_back(c);
        } else {
            // 替换该位置的末尾为更小的 c
            *it = c;
        }
    }
    return d.size();
}

int main() {
    int n;
    cin >> n;
    // 读取 n 个字符串（每个字符串一行，无空格）
    for (int i = 0; i < n; ++i) {
        string s;
        cin >> s;  // 直接输入，跳过前导空白
        cout << LIS(s) << endl;
    }
    return 0;
}
```
